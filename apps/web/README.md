# apps/web

The Meshploy web dashboard. A dark-only single-page app built with Vite, React 19, and TanStack Router.

---

## Stack

| | |
|---|---|
| Build tool | Vite 6 |
| Framework | React 19 (client-side only, no SSR) |
| Routing | TanStack Router (file-based, `src/routes/`) |
| Styling | Tailwind CSS v4 — CSS-first via `@tailwindcss/vite`, no config file |
| Components | shadcn/ui (Nova preset) using `@base-ui/react` primitives |
| State | Zustand 5 |
| Icons | lucide-react |

> **Note:** shadcn/ui components use `@base-ui/react`, **not** Radix UI. There is no `asChild` prop — use the `render` prop instead. See `AGENTS.md` for details.

## API client

All API calls go through `src/lib/api.ts`. In production the base URL is `""` (empty) — requests use relative paths (`/api/v1/...`) which Caddy routes to the API on port 4000. In development the base URL falls back to `http://localhost:4000`.

The runtime base URL is injected at container startup via `/config.js` (written by `docker-entrypoint.sh`).

## Dev commands

```bash
npm install
npm run dev       # Vite dev server at http://localhost:5173
npm run build     # Type-check + production build → dist/
npm run preview   # Preview the production build
```

TanStack Router's route tree (`src/routeTree.gen.ts`) is auto-generated by the Vite plugin — never edit it manually.

---

## Production build

The web app is served by nginx inside Docker. The `Dockerfile` has two stages:

1. **Builder** — runs `npm run build`, outputs to `dist/`
2. **Runtime** — nginx serves `dist/`, with `try_files` for SPA routing and `no-store` on `/config.js`

The `docker-entrypoint.sh` writes `/config.js` at startup:

```js
window.__MESHPLOY_CONFIG__ = { apiUrl: "" };
```

Build context is `apps/web/` only — no monorepo dependency.
